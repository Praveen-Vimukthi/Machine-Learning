## Introduction

**Natural Language Processing (NLP)** is a branch of computer science and machine learning that trains computers to process large amounts of human (natural) language data. In short — it is the ability of computers to understand human language.

Machine Learning algorithms learn from a pre-defined set of features from training data to produce output for test data. The core problem is that **ML algorithms cannot work on raw text directly**. We need **feature extraction techniques** to convert text into a matrix (or vector) of numerical features.

The two most popular methods covered here are:

| Method | Key Idea |
|--------|----------|
| **Bag-of-Words (BoW)** | Represent text as word frequency counts |
| **TF-IDF** | Weight words by how unique they are across the corpus |

---

## Why Feature Extraction?

```
Raw Text  →  [Feature Extraction]  →  Numerical Matrix  →  ML Algorithm  →  Prediction
```

Without feature extraction, algorithms have no way to measure similarity, train classifiers, or produce any output. Feature extraction is the **bridge between language and mathematics**.

## Bag-of-Words (BoW)

The **Bag-of-Words** model represents a text document as a **multiset of its words**, disregarding grammar and word order but keeping the frequency of each word. Think of it as emptying a document into a bag and counting how many times each word appears.

**Common use cases:**
- Text / document classification
- Sentiment analysis (e.g., detecting positive/negative reviews)
- Document similarity and clustering

> Words like `'fabulous'`, `'excellent'` indicate a positive review. Words like `'annoying'`, `'poor'` indicate a negative review.

---

### Step 1 — Text Preprocessing

Before building any model, clean the raw text:

1. **Lowercase** — convert all characters to lowercase so `Good` and `good` are treated as the same word
2. **Remove punctuation** — strip all punctuation marks and unnecessary symbols
3. **Tokenise** — split the cleaned text into individual words (tokens)

```python
import nltk
import string

def preprocess(text):
    text = text.lower()                          # lowercase
    text = text.translate(str.maketrans('', '', string.punctuation))  # remove punctuation
    tokens = nltk.word_tokenize(text)            # tokenise
    return tokens
```

---

### Step 2 — Build Vocabulary

Collect every **unique word** from the entire corpus (all documents combined). This vocabulary becomes the set of columns in your feature matrix.

**Example corpus:**

```
"good movie"
"not a good movie"
"did not like"
```

**Resulting vocabulary:**

```
{ good, movie, not, a, did, like }
```

---

### Step 3 — Text Vectorization

Create a matrix where:
- Each **row** = one document
- Each **column** = one word from the vocabulary
- Each **cell** = `1` if the word is present in the document, `0` if absent (binary BoW) — or the raw count (count BoW)

This process is called **Text Vectorization**.

| Review | good | movie | not | a | did | like |
|--------|------|-------|-----|---|-----|------|
| "good movie" | 1 | 1 | 0 | 0 | 0 | 0 |
| "not a good movie" | 1 | 1 | 1 | 1 | 0 | 0 |
| "did not like" | 0 | 0 | 1 | 0 | 1 | 1 |

---

### N-gram Extension

A major drawback of basic BoW is that **word order is completely lost**. The phrases "good movie" and "movie good" produce identical vectors.

**Solution — N-grams:** Instead of single words (unigrams), use sequences of N consecutive words (bigrams, trigrams, etc.) to partially preserve local word order.

| Bigram | "good movie" | "not a good movie" | "did not like" |
|--------|--------------|--------------------|----------------|
| good movie | 1 | 1 | 0 |
| not good | 0 | 1 | 0 |
| did not | 0 | 0 | 1 |

> ⚠️ **Warning:** N-grams produce very large, very sparse matrices. The number of possible bigrams grows much faster than unigrams. Always filter by frequency:
> - **Remove high-frequency N-grams** → these are usually stopwords (the, a, and)
> - **Remove very low-frequency N-grams** → these are usually typos or noise
> - **Keep medium-frequency N-grams** → these carry the most signal
>
> But even medium-frequency filtering can miss rare-but-important terms like `"Wi-Fi breaks"`. This is where **TF-IDF** helps.

---

### BoW Code Example

```python
import nltk

# Sample corpus
dataset = ['good movie', 'not a good movie', 'did not like']

# Build word frequency dictionary (manual BoW)
word2count = {}
for data in dataset:
    words = nltk.word_tokenize(data)   # tokenise each document
    for word in words:
        if word not in word2count.keys():
            word2count[word] = 1       # first occurrence
        else:
            word2count[word] += 1      # increment count

print(word2count)
# Output: {'good': 2, 'movie': 2, 'not': 2, 'a': 1, 'did': 1, 'like': 1}
```

**Using scikit-learn's `CountVectorizer` for the full feature matrix:**

```python
from sklearn.feature_extraction.text import CountVectorizer
import pandas as pd

corpus = ['good movie', 'not a good movie', 'did not like']

vectorizer = CountVectorizer()
X = vectorizer.fit_transform(corpus)

df = pd.DataFrame(X.todense(), columns=vectorizer.get_feature_names_out())
print(df)
```

---

### Issues with BoW

| Issue | Description |
|-------|-------------|
| **High dimensionality** | Feature space grows with vocabulary size, causing overfitting and slow computation |
| **Loss of word order** | "not good" and "good not" produce identical vectors |
| **No context** | Treats each word independently; cannot understand phrases or negations |
| **No semantic meaning** | "car" and "automobile" are treated as completely unrelated |
| **Stopword sensitivity** | Common words like "the", "and", "a" dominate the vector if not filtered |
| **Sparsity** | Most entries in the feature matrix are zero, wasting memory |

## TF-IDF Vectorizer

**TF-IDF** (Term Frequency — Inverse Document Frequency) is a statistical measure used in information retrieval and NLP. It reflects **how important a word is to a document relative to the entire corpus**.

The core idea:
> A word that appears **frequently in one document** but **rarely across the corpus** is more informative than a word that is common everywhere.

**Common use cases:**
1. Information retrieval and search engines
2. Document classification and text categorisation
3. Text summarisation
4. Feature extraction for ML algorithms

TF-IDF is composed of two parts: **TF** and **IDF**.

---

### Term Frequency (TF)

TF measures how **frequently a term appears within a single document**. It can be thought of as the probability of finding a word in that document.

**Formula:**

```
tf(wᵢ, rⱼ) = (Number of times wᵢ occurs in rⱼ) / (Total number of words in rⱼ)
```

**Log-normalisation variant** (reduces the impact of very high-frequency terms):

```
tf(t, d) = 1 + log(f_t,d)
```

Where `f_t,d` is the raw frequency of term `t` in document `d`.

**Example:** In the document `"good movie good"` (3 words):
- `tf("good", doc) = 2/3 = 0.67`
- `tf("movie", doc) = 1/3 = 0.33`

---

### Inverse Document Frequency (IDF)

IDF measures how **rare or common a term is across the whole corpus**. Words that appear in almost every document (like "the") get a low IDF score. Words that appear in only one or two documents get a high IDF score.

**Formula:**

```
idf(t, D) = log( |D| / |{d ∈ D : t ∈ d}| )
```

Where:
- `|D|` = total number of documents in the corpus
- `|{d ∈ D : t ∈ d}|` = number of documents that contain the term `t`

> Since the fraction inside the log is always ≥ 1, IDF is always ≥ 0.
> When a term appears in **all** documents, the ratio → 1, and log(1) = 0, making IDF → 0.

---

### Combined TF-IDF Score

```
tfidf(t, d, D) = tf(t, d) × idf(t, D)
```

| Scenario | TF | IDF | TF-IDF | Meaning |
|----------|----|-----|--------|---------|
| Word frequent in doc, rare in corpus | High | High | **High** | Very important |
| Word frequent in doc, common in corpus | High | Low | Low | Probably a stopword |
| Word rare in doc, rare in corpus | Low | High | Medium | Somewhat important |
| Word rare in doc, common in corpus | Low | Low | **Low** | Not important |

---

### Worked Example

**Corpus (3 documents):**
```
Doc 1: "good movie"
Doc 2: "not a good movie"
Doc 3: "did not like"
```

**TF-IDF scores for selected bigrams:**

| Document | "good movie" | "movie" | "did not" |
|----------|-------------|---------|-----------|
| Doc 1 | 1 × log(3/2) = **0.17** | 1 × log(3/2) = 0.17 | 0 × log(3/1) = 0 |
| Doc 2 | 1 × log(3/2) = **0.17** | 1 × log(3/2) = 0.17 | 0 × log(3/1) = 0 |
| Doc 3 | 0 × log(3/2) = 0 | 0 × log(3/2) = 0 | 1 × log(3/1) = **0.47** |

**Observation:** `"did not"` gets the highest score (0.47) because it is **rare** — appearing in only 1 of 3 documents. This is exactly the behaviour TF-IDF is designed for: surfacing terms that are locally important but globally rare.

### `TfidfVectorizer` — Full API Reference

From `sklearn.feature_extraction.text`. Converts a raw corpus into a TF-IDF feature matrix.

```python
from sklearn.feature_extraction.text import TfidfVectorizer

tfidf = TfidfVectorizer(
    min_df=1,
    max_df=1.0,
    ngram_range=(1, 1),
    max_features=None,
    stop_words=None,
    lowercase=True,
    analyzer='word',
    sublinear_tf=False,
    use_idf=True,
    smooth_idf=True,
    norm='l2',
    vocabulary=None,
    token_pattern=r"(?u)\b\w\w+\b",
    dtype=float
)
```

#### Parameter Reference

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `min_df` | `int` or `float` | `1` | Ignore terms with document frequency **below** this threshold. `int` = absolute count (e.g., `2` means ignore terms appearing in fewer than 2 docs). `float` = proportion of corpus (e.g., `0.01` = ignore if in fewer than 1% of docs). Use to remove typos and rare noise. |
| `max_df` | `int` or `float` | `1.0` | Ignore terms with document frequency **above** this threshold. `float` = proportion (e.g., `0.5` = ignore if in more than 50% of docs). Effectively removes corpus-specific stopwords automatically. |
| `ngram_range` | `tuple (min_n, max_n)` | `(1, 1)` | Range of N-gram sizes to extract. `(1,1)` = unigrams only. `(1,2)` = unigrams + bigrams. `(2,2)` = bigrams only. Larger ranges increase feature space significantly. |
| `max_features` | `int` or `None` | `None` | Build vocabulary using only the top N features ordered by term frequency across the corpus. Set this to reduce dimensionality (e.g., `max_features=5000`). |
| `stop_words` | `'english'`, `list`, or `None` | `None` | Words to exclude from the vocabulary. `'english'` uses scikit-learn's built-in English stopword list. Pass a custom list for domain-specific filtering. |
| `lowercase` | `bool` | `True` | Convert all characters to lowercase before tokenising. Ensures `Good` and `good` are treated as the same token. |
| `analyzer` | `'word'`, `'char'`, or `'char_wb'` | `'word'` | Unit of analysis. `'word'` = word N-grams (standard). `'char'` = character N-grams (useful for morphologically rich languages or handling typos). `'char_wb'` = character N-grams only within word boundaries. |
| `sublinear_tf` | `bool` | `False` | Apply sublinear TF scaling: replace TF with `1 + log(TF)`. Reduces the disproportionate impact of very high raw frequencies. Recommended for long documents. |
| `use_idf` | `bool` | `True` | Enable inverse-document-frequency reweighting. If `False`, the vectoriser returns a plain TF matrix with no IDF weighting. |
| `smooth_idf` | `bool` | `True` | Add 1 to all document frequencies before computing IDF to prevent zero-division errors: `idf = log((1+n)/(1+df)) + 1`. Recommended to leave as `True`. |
| `norm` | `'l1'`, `'l2'`, or `None` | `'l2'` | Row normalisation applied to output vectors. `'l2'` makes each document vector unit-length (most common). `'l1'` normalises so values sum to 1. `None` disables normalisation entirely. |
| `vocabulary` | `dict` or `None` | `None` | Provide a fixed vocabulary mapping terms to feature indices. If set, the vectoriser only uses these terms and ignores all others. Useful for applying a pre-trained vocabulary to new data. |
| `token_pattern` | `str` (regex) | `r"(?u)\b\w\w+\b"` | Regular expression defining what constitutes a valid token. The default selects words of 2 or more alphanumeric characters. Ignored when `analyzer != 'word'`. |
| `dtype` | numpy dtype | `float64` | Data type of the values in the output matrix. Use `float32` to save memory on large corpora. |

---

### Key Methods & Attributes

#### Methods

| Method | Returns | Description |
|--------|---------|-------------|
| `.fit(corpus)` | `self` | Learn the vocabulary and compute IDF weights from the corpus. Does **not** transform the data. |
| `.transform(corpus)` | sparse matrix | Transform documents to a TF-IDF feature matrix using the already-learned vocabulary. Use on test data after fitting on training data. |
| `.fit_transform(corpus)` | sparse matrix | Fit and transform in a single step. Equivalent to `.fit().transform()` but more efficient. Use only on training data. |
| `.get_feature_names_out()` | `array of str` | Returns the list of feature names (vocabulary terms) in the same order as the matrix columns. |
| `.inverse_transform(X)` | `list of arrays` | Given a TF-IDF matrix, return the original terms for each row that have non-zero coefficients. Useful for inspecting what each document contains. |

#### Attributes (available after `.fit()`)

| Attribute | Type | Description |
|-----------|------|-------------|
| `.vocabulary_` | `dict` | Mapping of terms to their column index in the feature matrix. e.g., `{'good': 0, 'movie': 1, ...}` |
| `.idf_` | `array` | The IDF weight for each term in the vocabulary, in column order. Higher value = rarer term. |
| `.stop_words_` | `set` | Terms that were ignored because of `max_df`, `min_df`, or the `stop_words` setting. |

### TF-IDF Code Example

```python
from sklearn.feature_extraction.text import TfidfVectorizer
import pandas as pd

# Sample corpus
corpus = [
    'good movie',
    'not a good movie',
    'did not like'
]

# Initialise with common settings
tfidf = TfidfVectorizer(
    min_df=1,              # keep terms in at least 1 document
    max_df=0.9,            # drop terms appearing in more than 90% of docs
    ngram_range=(1, 2),    # use both unigrams and bigrams
    stop_words='english',  # filter English stopwords
    sublinear_tf=True,     # apply log-TF scaling
    norm='l2'              # L2-normalise output rows
)

# Fit and transform
X = tfidf.fit_transform(corpus)

# View as a readable DataFrame
df = pd.DataFrame(
    X.todense(),
    columns=tfidf.get_feature_names_out()
)
print(df)

# Inspect vocabulary and IDF weights
print("\nVocabulary:", tfidf.vocabulary_)
print("IDF weights:", tfidf.idf_)

# Transform new, unseen documents (use transform, not fit_transform)
new_docs = ['great film', 'terrible movie']
X_new = tfidf.transform(new_docs)
```

### Issues with TF-IDF

| Issue | Description |
|-------|-------------|
| **High dimensionality** | Like BoW, the feature space can be very large, causing overfitting and slow computation |
| **Loss of word order** | TF-IDF considers individual terms, not sequences or syntax |
| **No semantic meaning** | "car" and "automobile" are unrelated in TF-IDF; it has no concept of synonyms |
| **No word associations** | Cannot capture that "not good" is the opposite of "good" |
| **Domain dependence** | IDF scores are corpus-specific; a model trained on news articles may not generalise to medical text |
| **Stopword sensitivity** | Without filtering, stopwords can still dominate if they happen to be rare in the specific corpus |

## Quick Comparison

| Feature | BoW | TF-IDF |
|---------|-----|--------|
| Preserves word order | ❌ No | ❌ No |
| Handles rare but important terms | ❌ No | ✅ Yes (via IDF) |
| Down-weights common terms | ❌ No | ✅ Yes (IDF → 0) |
| Sparse output | ✅ Yes | ✅ Yes |
| Captures word meaning | ❌ No | ❌ No |
| Complexity | Low | Medium |
| Best for | Simple classification | Retrieval, ranking, rare term detection |

## When to Use What

```
Start here
    │
    ├─── Need to detect rare/specific terms? ──── YES ──→ TF-IDF
    │
    ├─── Simple classification with many labelled docs? ──→ BoW (fast baseline)
    │
    ├─── Need semantic understanding? ──────────────────→ Word2Vec / GloVe / BERT
    │
    └─── Need phrase-level features? ───────────────────→ BoW / TF-IDF with ngram_range=(1,2)
```

> For production NLP systems requiring semantic understanding, consider **word embeddings** (`Word2Vec`, `GloVe`) or **transformer models** (`BERT`, `sentence-transformers`). BoW and TF-IDF are excellent baselines and still widely used for interpretable, fast pipelines.
